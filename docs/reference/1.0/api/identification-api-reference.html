Identification

Sends identification information to an identification server.

Client ⇔ NT-REST

Loopback connections only. Authenticated connections only.

Request

Method URL

POST

http://localhost.ntre.st/nt-rest/v1/identification

Headers

Name Is mandatory Expected value

Content-Type

Yes

“text/json; charset=UTF-8” or
“application/json; charset=UTF-8” or
“application/bson” or
“application/cbor” or
“application/msgpack” or
“application/ubjson”

Cookie

Yes

Session-ID=<session-id>

X-Session-ID

<session-id>

Body

No request body is required. However, any non-empty request body received from client is forwarded from client to identification server as is, without any processing.

Response

Status

Status code returned by identification server is forwarded from identification server to client as is, without any processing. Recommended status codes are:

200

Identification succeeded.

403

Identification is forbidden.

Client Example

JavaScript

fetch(
    'http://localhost.ntre.st/nt-rest/v1/identification',
    {
        method: 'POST',
        cache: 'no-cache',
        mode: 'cors',
        credentials: 'include',
        headers: {
            'Content-Type': 'application/json; charset=UTF-8',
            'X-Session-ID': '<session-id>'
        },
        redirect: 'error',
        referrerPolicy: 'origin-when-cross-origin',
        body: '<additional-data>'
    }
);
